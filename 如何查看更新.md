# 🎨 如何查看UI更新

## 快速开始

### 1. 确保服务器正在运行

```bash
python run.py
```

### 2. 访问新的科技风格页面

打开浏览器，访问以下地址：

#### 🏠 首页（科技风格）
```
http://localhost:8000/
```

**特点**:
- 深色科技主题
- 霓虹蓝色调
- 动态背景动画
- 快递查询功能
- 导航到客户/员工登录

#### 👤 客户登录
```
http://localhost:8000/customer/login.html
```

**特点**:
- 科技风格登录界面
- 用户名/邮箱登录
- 记住我功能
- 注册入口

#### ✨ 客户注册
```
http://localhost:8000/customer/register.html
```

**特点**:
- 完整的注册表单
- 实时密码验证
- 服务条款确认

#### 🔐 员工登录
```
http://localhost:8000/admin/login.html
```

**特点**:
- 科技风格管理员登录
- 与整体风格统一
- 实际可用（连接到后端API）

**测试账号**:
- 用户名: `admin`
- 密码: `admin123`

## 🧪 测试快递查询功能

### 在首页测试

1. 访问 http://localhost:8000/
2. 在搜索框输入任意快递单号（例如：`YT1234567890`）
3. 点击"查询"按钮
4. 查看返回的物流信息

**现在会显示**:
- ✅ 快递公司信息
- ✅ 物流状态
- ✅ 完整的物流轨迹时间线
- ✅ 每个节点的时间、地点、描述
- ✅ 理货信息（如果有）

**之前的问题**:
- ❌ 只返回简单的"ok"字段

## 📱 测试响应式设计

### 桌面端
- 打开浏览器，正常访问

### 移动端
1. 按 `F12` 打开开发者工具
2. 点击设备工具栏图标（或按 `Ctrl+Shift+M`）
3. 选择移动设备（如 iPhone 12）
4. 查看页面自适应效果

## 🐛 已知问题

### 客户系统API未实现
- **现象**: 客户登录/注册提交后显示"功能开发中"
- **原因**: 后端API尚未实现
- **影响**: 仅前端界面可用，无法实际注册/登录
- **解决**: 需要实现后端API和数据库表

### 快递查询可能失败
- **现象**: 某些单号查询失败
- **原因**: 快递100 API限制或单号不存在
- **影响**: 显示错误提示
- **正常**: 这是预期行为
